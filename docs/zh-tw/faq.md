# 常見問題

## 一般問題

### 什麼是 BEE？

BEE（Backend Engineering Essentials，後端工程精要）是一組編號的後端工程指引。完整介紹請見 [BEE-0](0)。

### BEE 的目標讀者是誰？

擁有 1-3 年經驗的後端工程師，以及正在入門後端開發的人。BEE 建立概念基礎，而非特定廠商的教學。

### BEE 是否針對特定語言或框架？

不是。BEE 原則與廠商和語言無關。當需要用特定技術來說明概念時，會明確標註為範例，而非建議。

### BEE 和 ADP、DDP 的關係是什麼？

BEE 廣泛涵蓋後端工程。ADP（API 設計原則）深入 API 設計。DDP（資料庫設計原則）深入資料庫設計。主題重疊時，BEE 會連結到 ADP 和 DDP 做深入探討。

### 我可以提議新的 BEE 嗎？

可以。請遵循 [BEE-0](0) 中描述的結構並提交 pull request。

## 內容問題

### 為什麼使用 RFC 2119 關鍵字（MUST、SHOULD、MAY）？

這些關鍵字提供精確的指引層級。MUST 表示該原則不可協商。SHOULD 表示除非有記錄的理由，否則應遵循。MAY 表示可選。

### BEE 編號如何分配？

BEE 按類別範圍編號。完整對應表請見 [BEE-0](0) 的類別表。

### 如果我的情況需要偏離 BEE 怎麼辦？

記錄原因。BEE 是指引，不是法律。目標是做出有根據的決定，而非盲目遵循規則。
