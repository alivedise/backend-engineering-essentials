# [BEE-0] BEE 總覽

:::info
BEE（Backend Engineering Essentials，後端工程精要）是一組與廠商和語言無關的後端工程指引與最佳實踐。
:::

## 背景

後端工程涵蓋廣闊的領域 -- 從認證和網路到分散式系統和可觀測性。工程師通常零散地學習這些概念，透過散落的部落格文章、口耳相傳的知識，或痛苦的生產事故。BEE 提供一組結構化、編號的原則，幫助建立連貫的後端工程心智模型。

## 目的

- 建立後端工程討論的共同詞彙
- 提供可操作的、廠商中立的指引，處理常見的後端決策
- 作為進入後端開發的工程師的入門資源
- 填補「我會寫語法」和「我理解系統」之間的落差

## 如何閱讀 BEE

每篇 BEE 遵循一致的結構：

- **背景** -- 為什麼這個原則重要
- **原則** -- 核心指引（使用 RFC 2119 關鍵字：MUST、SHOULD、MAY）
- **圖解** -- 有助理解的圖表
- **範例** -- 具體的、廠商中立的範例
- **常見錯誤** -- 應避免的反模式
- **相關 BEE** -- 其他原則的交叉引用
- **參考資料** -- 外部資源的深入學習

## 類別

### 基礎層 (0-89)

| 範圍 | 類別 | 焦點 |
|------|------|------|
| 0-9 | BEE 總覽 | 目的、術語表、後設 |
| 10-29 | 認證與授權 | 身份、存取控制、令牌、會話 |
| 30-49 | 安全基礎 | OWASP、輸入驗證、密鑰、密碼學 |
| 50-69 | 網路基礎 | TCP/IP、DNS、HTTP、TLS、負載平衡 |
| 70-89 | API 設計與通訊協定 | REST、gRPC、GraphQL、版本控制、分頁 |

### 架構與資料層 (100-179)

| 範圍 | 類別 | 焦點 |
|------|------|------|
| 100-119 | 架構模式 | 單體、微服務、DDD、CQRS、六角形 |
| 120-139 | 資料儲存與資料庫基礎 | SQL vs NoSQL、索引、複製、分片 |
| 140-159 | 資料建模與結構設計 | ER 建模、正規化、序列化 |
| 160-179 | 交易與資料完整性 | ACID、隔離等級、saga、冪等性 |

### 執行時期層 (200-279)

| 範圍 | 類別 | 焦點 |
|------|------|------|
| 200-219 | 快取 | 失效、淘汰、分散式快取、HTTP 快取 |
| 220-239 | 訊息與事件驅動 | 佇列、發布/訂閱、交付保證、事件溯源 |
| 240-259 | 並行與非同步 | 執行緒、鎖、非同步 I/O、工作池 |
| 260-279 | 韌性與可靠性 | 斷路器、重試、逾時、限流 |

### 工程實踐層 (300-379)

| 範圍 | 類別 | 焦點 |
|------|------|------|
| 300-319 | 效能與可擴展性 | 估算、擴展、剖析、CDN |
| 320-339 | 可觀測性 | 日誌、指標、追蹤、SLO、告警 |
| 340-359 | 測試策略 | 測試金字塔、整合、契約、負載測試 |
| 360-379 | CI/CD 與 DevOps | CI、部署策略、IaC、功能旗標 |
