# BEE Numbering and Category Restructure Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate the BEE catalog from integer-only IDs to category-blocked IDs, and from URL-as-integer (`/596`) to semantic URL (`/api-design/graphql-http-caching`), with full backward-compat redirects, in a single atomic commit produced by a tested migration script.

**Architecture:** A Node.js migration script (`scripts/migrate-bee-numbering.mjs`) is the single tool that performs the migration. It is built TDD: pure functions for slug generation, ID assignment, mapping construction, and cross-reference rewriting each get unit tests before implementation, using Node 20's built-in test runner. The script orchestrates renames, frontmatter updates, link rewrites, config patches, and build validation; it commits a single atomic change only after `pnpm docs:build` passes. Backward-compat is handled by an HTML meta-refresh stub generator wired into VitePress as a `buildEnd` hook. Bilingual parity is guaranteed by processing EN and zh-TW with the same mapping table in lockstep.

**Tech Stack:** Node.js 20 (matches `.github/workflows/deploy.yml`), Node's built-in `node:test` runner and `node:assert/strict`, `gray-matter` (already a project dep) for frontmatter parsing, VitePress 1.3.1, pnpm 8.15.5, JavaScript (ESM `.mjs`), git for atomic file rename history.

## File Structure

The migration script is decomposed into pure-function modules so each module can be tested in isolation. Orchestration is the only impure code.

```
scripts/
├── migrate-bee-numbering.mjs          # Orchestrator (impure: filesystem, git, build)
├── migrate-bee-numbering.test.mjs     # End-to-end test using the orchestrator on a temp fixture
└── lib/
    ├── slug.mjs                       # Pure: titleToSlug(title) → string
    ├── slug.test.mjs                  # Tests for slug.mjs
    ├── blocks.mjs                     # Pure: BLOCK_ALLOCATION constant + assignNewIds(category, articles) → mapping[]
    ├── blocks.test.mjs                # Tests for blocks.mjs
    ├── mapping.mjs                    # Pure: buildMapping(allArticles) → MappingEntry[]
    ├── mapping.test.mjs               # Tests for mapping.mjs
    ├── crossref.mjs                   # Pure: rewriteCrossRefs(markdownText, mapping) → string
    ├── crossref.test.mjs              # Tests for crossref.mjs
    ├── frontmatter.mjs                # Pure: updateFrontmatter(rawMd, newId, slug) → newRawMd
    ├── frontmatter.test.mjs           # Tests for frontmatter.mjs
    └── redirects.mjs                  # Pure: renderRedirectStub(oldId, newPath, basePath, locale, title) → string

migration/
└── bee-id-mapping.json                # Generated by script, committed alongside migration

docs/superpowers/
└── MIGRATION-NOTE.md                  # Created by script

docs/.vitepress/config/
├── index.mjs                          # Patched: rewrites + buildEnd hook for redirects
├── en.js                              # Patched: sidebar generator uses slug + new URL format
└── zh-tw.js                           # Patched: same as en.js
```

**Why this decomposition:**
- Pure functions are testable without filesystem or git. Each `lib/*.mjs` module exports one or two named functions.
- The orchestrator is the only place that touches the filesystem, runs git commands, or invokes the build. It composes the pure functions.
- Tests live next to the code they test (`*.test.mjs` co-located).
- `BLOCK_ALLOCATION` constant lives in `blocks.mjs` so it's importable by both the script and the tests.

```javascript
// titleToSlug: derive a URL-safe English slug from an article title per spec §4.
//
// Rules:
//   - lowercase ASCII
//   - hyphens between words, no underscores or spaces
//   - drop leading articles (a, an, the)
//   - drop punctuation (, : ' " ? ! ( ) " ")
//   - preserve numerics joined with hyphens
//   - editorial shortenings for known patterns (in APIs → api-, /1.1 versions etc.)
//
// The function applies the rules deterministically. For titles that need
// editorial shortening (rare), the SHORTENING_RULES map is consulted first.

const SHORTENING_RULES = new Map([
  ['HTTP/1.1, HTTP/2, HTTP/3', 'http-versions'],
  ['Idempotency in APIs', 'api-idempotency'],
  ['The N+1 Query Problem and Batch Loading', 'n-plus-1-query-batching'],
  ['RBAC vs ABAC Access Control Models', 'rbac-vs-abac'],
  ['OAuth 2.0 and OpenID Connect', 'oauth-openid-connect'],
  ['"Webhooks" and Callback Patterns', 'webhooks-callback-patterns'],
]);

const LEADING_ARTICLES = new Set(['a', 'an', 'the']);

export function titleToSlug(title) {
  if (typeof title !== 'string' || title.trim().length === 0) {
    throw new Error('title must be non-empty');
  }

  if (SHORTENING_RULES.has(title)) {
    return SHORTENING_RULES.get(title);
  }

  let s = title.toLowerCase();

  // Strip punctuation but keep hyphens and alphanumerics and spaces and slashes
  // (slashes will become hyphens; quotes and other punct go).
  s = s.replace(/[,.:;!?()'"`"""''‐]/g, '');

  // Replace slashes with hyphens
  s = s.replace(/\//g, '-');

  // Collapse whitespace to single hyphen
  s = s.trim().replace(/\s+/g, '-');

  // Collapse multiple hyphens
  s = s.replace(/-{2,}/g, '-');

  // Drop leading article if present
  const tokens = s.split('-');
  if (LEADING_ARTICLES.has(tokens[0])) {
    tokens.shift();
  }

  return tokens.join('-');
}
```

```javascript
// assignNewIds: given a source folder name and an array of articles
// (each with { current_id, title }), return new array with new_id assigned
// in current_id ascending order starting from the block's start.
//
// Throws if the source folder has no block allocation, or if the article
// count exceeds the block's size.
export function assignNewIds(sourceFolder, articles) {
  const block = blockForSource(sourceFolder);
  if (!block) {
    throw new Error(`no block allocation for source: ${sourceFolder}`);
  }

  const blockSize = block.end - block.start + 1;
  if (articles.length > blockSize) {
    throw new Error(
      `${articles.length} articles exceeds block size ${blockSize} for ${block.slug}`
    );
  }

  const sorted = [...articles].sort((a, b) => a.current_id - b.current_id);
  return sorted.map((article, i) => ({
    ...article,
    new_id: block.start + i,
  }));
}
```

```javascript
import matter from 'gray-matter';

// updateFrontmatter: given raw markdown text, a new integer id, and a slug,
// return new markdown text with frontmatter id replaced and slug added.
// All other frontmatter fields preserved. Body untouched.
//
// Throws if input has no frontmatter.
export function updateFrontmatter(rawMd, newId, slug) {
  const parsed = matter(rawMd);
  if (!parsed.matter || parsed.matter.trim().length === 0) {
    throw new Error('no frontmatter found in input');
  }

  const newData = {
    ...parsed.data,
    id: newId,
    slug,
  };

  // gray-matter's stringify writes YAML frontmatter back out.
  return matter.stringify(parsed.content, newData);
}
```

```javascript
import { titleToSlug } from './slug.mjs';
import { blockForSource, assignNewIds } from './blocks.mjs';

// buildMapping: given a flat array of articles
//   [{ source_folder, current_id, title, en_path, zh_path }],
// return a mapping array with new_id, slug, category_new, new_path_en, new_path_zh.
//
// Articles are grouped by source folder, renumbered via assignNewIds
// (which honors the merged Databases → data-storage rule), and slugged.
export function buildMapping(articles) {
  // Group by source folder
  const byFolder = new Map();
  for (const a of articles) {
    if (!byFolder.has(a.source_folder)) {
      byFolder.set(a.source_folder, []);
    }
    byFolder.get(a.source_folder).push(a);
  }

  const mapping = [];
  // Track which destination block each source maps into to handle merges
  const idsByBlock = new Map();
  for (const [source, group] of byFolder) {
    const block = blockForSource(source);
    if (!block) {
      throw new Error(`no block for source folder: ${source}`);
    }
    if (!idsByBlock.has(block.slug)) {
      idsByBlock.set(block.slug, []);
    }
    idsByBlock.get(block.slug).push(...group.map(a => ({ ...a, _block: block })));
  }

  // For each destination block, sort and assign IDs
  for (const [blockSlug, items] of idsByBlock) {
    const block = items[0]._block;
    const sorted = [...items].sort((a, b) => a.current_id - b.current_id);
    const blockSize = block.end - block.start + 1;
    if (sorted.length > blockSize) {
      throw new Error(
        `${sorted.length} articles exceed block size ${blockSize} for ${blockSlug}`
      );
    }
    sorted.forEach((article, i) => {
      const slug = titleToSlug(article.title);
      const newId = block.start + i;
      mapping.push({
        current_id: article.current_id,
        new_id: newId,
        title: article.title,
        slug,
        category_old: article.source_folder,
        category_new: blockSlug,
        old_path_en: article.en_path,
        new_path_en: `docs/en/${blockSlug}/${slug}.md`,
        old_path_zh: article.zh_path,
        new_path_zh: `docs/zh-tw/${blockSlug}/${slug}.md`,
      });
    });
  }

  return mapping;
}

// validateUniqueSlugs: given a mapping array, throw if any (category_new, slug)
// pair is duplicated.
export function validateUniqueSlugs(mapping) {
  const seen = new Set();
  for (const entry of mapping) {
    const key = `${entry.category_new}/${entry.slug}`;
    if (seen.has(key)) {
      throw new Error(`duplicate slug in ${entry.category_new}: ${entry.slug}`);
    }
    seen.add(key);
  }
}
```

```javascript
// rewriteCrossRefs: given markdown text, a mapping array, and the
// current article's category slug (so we can compute relative paths),
// return new markdown text with all [BEE-X](path/X.md) links rewritten
// to [BEE-NEW](relative-path/new-slug.md).
//
// Pattern matched: [BEE-<digits>](<path-prefix>/?<digits>.md)
//
// Path-prefix can be:
//   - empty (intra-category): `[BEE-70](70.md)`
//   - `../<category-folder>/` (cross-category): `[BEE-205](../Caching/205.md)`
//   - any other directory traversal containing the category name
//
// Unknown IDs (not in mapping) are left unchanged.
export function rewriteCrossRefs(markdownText, mapping, currentCategorySlug) {
  const idToEntry = new Map(mapping.map(m => [m.current_id, m]));

  // Match [BEE-<id>](<path>) where <path> ends in <id>.md
  // Use a non-greedy capture for the path to avoid eating across links.
  const pattern = /\[BEE-(\d+)\]\(([^)]*?)(\d+)\.md\)/g;

  return markdownText.replace(pattern, (match, beeId, pathPrefix, fileId) => {
    const id = parseInt(beeId, 10);
    const fileIdInt = parseInt(fileId, 10);
    if (id !== fileIdInt) {
      // Mismatch between [BEE-X] and (X.md) — leave alone, possibly intentional.
      return match;
    }

    const entry = idToEntry.get(id);
    if (!entry) {
      // Unknown ID — leave intact.
      return match;
    }

    // Compute new path. If target is in same category as current, use bare filename.
    // Otherwise, use ../{target-category-slug}/{slug}.md
    let newPath;
    if (entry.category_new === currentCategorySlug) {
      newPath = `${entry.slug}.md`;
    } else {
      newPath = `../${entry.category_new}/${entry.slug}.md`;
    }

    return `[BEE-${entry.new_id}](${newPath})`;
  });
}
```

## Task 15: Add buildEnd hook for redirect stub generation

**Files:**
- Modify: `docs/.vitepress/config/index.mjs`

- [ ] **Step 1: Add the buildEnd hook to the base config**

In `docs/.vitepress/config/index.mjs`, modify the `defineConfig` block to add a `buildEnd` callback:

```javascript
import { defineConfig } from 'vitepress';
import { withPwa } from '@vite-pwa/vitepress';
import { withMermaid } from 'vitepress-plugin-mermaid';
import { readFileSync, writeFileSync, mkdirSync, existsSync } from 'node:fs';
import { join, dirname, resolve } from 'node:path';

import { shared } from './shared';
import { en } from './en';
import { zhTW } from './zh-tw';

const REPO_ROOT = resolve(import.meta.dirname, '../../..');
const MAPPING_PATH = join(REPO_ROOT, 'migration/bee-id-mapping.json');
const BASE_PATH = '/backend-engineering-essentials';

function renderRedirectStubInline({ destPath, title }) {
  return `<!DOCTYPE html>
<html>
<head>
<meta charset="utf-8">
<meta http-equiv="refresh" content="0; url=${destPath}">
<link rel="canonical" href="${destPath}">
<title>Redirecting to ${title}</title>
</head>
<body>
<p>This page has moved to <a href="${destPath}">${destPath}</a>.</p>
</body>
</html>
`;
}

function emitRedirectStubs(siteConfig) {
  if (!existsSync(MAPPING_PATH)) {
    console.warn('[buildEnd] no mapping file at', MAPPING_PATH, '— skipping redirect stubs');
    return;
  }
  const mapping = JSON.parse(readFileSync(MAPPING_PATH, 'utf-8'));
  const distRoot = siteConfig.outDir;
  let stubCount = 0;
  for (const entry of mapping) {
    const newSemanticPath = `/${entry.category_new}/${entry.slug}`;
    // EN stub
    const enStubDir = join(distRoot, String(entry.current_id));
    mkdirSync(enStubDir, { recursive: true });
    writeFileSync(
      join(enStubDir, 'index.html'),
      renderRedirectStubInline({
        destPath: `${BASE_PATH}${newSemanticPath}`,
        title: entry.title,
      })
    );
    // zh-TW stub
    const zhStubDir = join(distRoot, 'zh-tw', String(entry.current_id));
    mkdirSync(zhStubDir, { recursive: true });
    writeFileSync(
      join(zhStubDir, 'index.html'),
      renderRedirectStubInline({
        destPath: `${BASE_PATH}/zh-tw${newSemanticPath}`,
        title: entry.title,
      })
    );
    stubCount += 2;
  }
  console.log(`[buildEnd] wrote ${stubCount} redirect stubs to ${distRoot}`);
}

const baseConfig = defineConfig({
  ...shared,
  rewrites: {
    'en/:category/:page': ':category/:page',
    'en/list.md': 'list.md',
    'en/faq.md': 'faq.md',
    'zh-tw/:category/:page': 'zh-tw/:category/:page',
  },
  locales: {
    root: { label: 'English', ...en },
    'zh-tw': { label: '繁體中文', ...zhTW },
  },
  buildEnd: (siteConfig) => {
    emitRedirectStubs(siteConfig);
  },
});

const withMermaidConfig = withMermaid(baseConfig);

export default withPwa(withMermaidConfig);
```

Expected: loads without throwing. (The hook itself only runs during build, not at import.)
